## Contexto por país — tickets y terminología

### Argentina
- Restaurante: $2.500–$8.000 ARS por cubierto
- Cafetería: $800–$2.500 ARS por consumo
- Terminología: "carta", "cubierto", "mesa", "mozo", "delivery", "pedido", "propina", "factura"
- Canal dominante: WhatsApp + Instagram DMs
- Plataformas delivery: PedidosYa, Rappi

### México (Robert — mercado objetivo)
- Restaurante: $120–$500 MXN por persona
- Cafetería especialidad: $80–$200 MXN por consumo
- Terminología: "menú", "orden", "mesero", "cuenta", "para llevar", "a domicilio", "antojo"
- Canal dominante: WhatsApp + Facebook + Instagram
- Plataformas delivery: Rappi, DiDi Food, Uber Eats
- **Particularidad MX**: el cliente pide mucho por WhatsApp directo, evita apps de terceros
- Horarios: comida fuerte 14:00–16:00, cena 20:00–22:00

### Colombia
- Restaurante: $25.000–$80.000 COP por persona
- Terminología: "pedido", "domicilio", "mesero", "la cuenta", "a la orden"
- Plataformas delivery: Rappi, iFood
- Canal: WhatsApp muy dominante para pedidos directos

### Chile

- Terminología: "menú", "plato", "mozo", "la cuenta", "para llevar", "delivery"

## Buyer persona del dueño gastronómico

### El que "no da más"
- Dueño que atiende él mismo: cocina, caja, WhatsApp y redes
- Pain principal: "me llegan mensajes a las 2am y no puedo responder"
- Quiere: que alguien atienda sin que él tenga que estar
- Trigger de venta: "imaginate que el bot toma el pedido y vos solo cocinás"

### El que quiere crecer
- Tiene local funcionando, quiere escalar o abrir segunda sucursal
- Pain: no tiene datos de sus clientes, no sabe qué vende más
- Quiere: CRM, métricas, fidelización
- Trigger: "¿cuántos clientes repetidores tenés? ¿Lo sabés?"

### El que acaba de abrir
- Nuevo local, muy digitalmente activo en IG
- Pain: poca automatización, todo manual
- Quiere: verse profesional, responder rápido
- Trigger: "dale un look profesional a tus respuestas desde el día 1"

## Preguntas clave para calificar un lead gastronómico

1. ¿Qué tipo de local tenés? (restaurante, café, delivery, etc.)
2. ¿Recibís pedidos o consultas por WhatsApp ahora?
3. ¿Cuántos mensajes por día aproximadamente no podés responder a tiempo?
4. ¿Manejás delivery, mesa, o ambos?
5. ¿Tenés menú digital? ¿Necesitás reservas?
6. ¿Cuántas personas atienden el WhatsApp hoy?

## Flujo conversacional del bot — patrón gastronómico

```
inicio → bienvenida con nombre del local
  → menú principal:
    1. Ver menú / carta
    2. Hacer un pedido
    3. Reservar una mesa
    4. Consultar horarios / ubicación
    5. Hablar con el equipo
  → según elección:
    - Pedido → tipo → detalles → confirmación → asesor/cocina
    - Reserva → fecha → hora → cantidad personas → nombre → confirmar
    - Menú → enviar link o imagen del menú del día
    - Info → respuesta automática con horarios/dirección
  → cierre → "¿Algo más en lo que pueda ayudarte?"
```

## Stack técnico habitual para este nicho

| Componente | Solución |
|------------|----------|
| Bot de pedidos/consultas | FastAPI + YCloud/Meta + Gemini |

| CRM clientes | Airtable: Clientes, Pedidos, Reservas |

## Integración Airtable — campos estándar para gastronomía

### Tabla Menú
```
Nombre (text), Categoría (singleSelect: Entradas/Principales/Postres/Bebidas/Combos),
Descripción (longText), Precio (currency ARS), Disponible (checkbox),
Imagen (attachment), Tiempo_Prep (number, minutos), Alérgenos (multiSelect)
```

### Tabla Pedidos
```
Número_Pedido (autonumber), Cliente_Nombre (text), Cliente_Tel (text),
Items (longText), Total (currency), Tipo (singleSelect: Mesa/Delivery/Takeaway),
Estado (singleSelect: Recibido/En_Preparación/Listo/Entregado),
Dirección_Entrega (text), Fecha_Hora (dateTime), Notas (text)
```

### Tabla Reservas
```
Nombre (text), Teléfono (text), Fecha (date), Hora (text),
Personas (number), Estado (singleSelect: Confirmada/Cancelada/Lista),
Notas (text)
```

### Tabla Clientes Frecuentes
```
Nombre (text), Teléfono (text), Email (email),
Pedidos_Count (number), Último_Pedido (date),
Favoritos (longText), Cumpleaños (date), Puntos_Fidelidad (number)
```

## Personalización rápida del demo para un cliente

Cuando llegue un cliente gastronómico nuevo, para adaptar el demo en `DEMOS/GASTRONOMIA/gastronomia.html`:

1. Cambiar nombre del local en el header
2. Cargar su menú real en la tabla Airtable del demo
3. Ajustar colores según brandbook del cliente
4. Cambiar el número de WhatsApp de demo al suyo
5. Mostrar el panel en vivo durante la reunión de ventas
