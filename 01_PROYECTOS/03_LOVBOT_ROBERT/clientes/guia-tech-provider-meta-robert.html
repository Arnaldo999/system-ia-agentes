Tech Provider de Meta
Guía paso a paso

Verificá y completá tu alta como proveedor de tecnología para onboardear clientes con WhatsApp Coexistence de forma 100% oficial.

👤 Para: Robert Bazán — Lovbot.mx 📅 Marzo 2026

¿Qué te habilita ser Tech Provider?

  • ✅ Vincular números WhatsApp Business de tus clientes a tu propia app de Meta — sin Evolution API ni hacks
  • ✅ Usar WhatsApp Coexistence — el cliente sigue usando su WA Business en el celu Y el bot corre al mismo tiempo
  • ✅ Onboardear múltiples clientes de forma oficial y gratuita
  • ✅ Línea de soporte directo con Meta para resolver bloqueos
1
Verificar si ya sos Tech Provider
▼
  1. Entrá a developers.facebook.com
  2. Seleccioná tu aplicación (la que usás para WhatsApp)
  3. En el panel buscá la sección "Configuración" o "WhatsApp"
  4. Si ves el mensaje "Tu empresa se ha verificado como proveedor de tecnología" → ✅ ya estás dado de alta, saltá al Paso 5
  5. Si no aparece ese mensaje → continuar con el Paso 2
2
Verificar tu empresa en Meta Business Manager
▼
  1. Entrá a business.facebook.com
  2. Menú izquierdo → Configuración (⚙️)
  3. Dentro de configuración → Centro de seguridad o Verificación del negocio
  4. Seguí el proceso de verificación (requiere: nombre legal, dirección, RFC/CUIT y documento oficial)
  5. Esperá aprobación: puede tardar 1 a 5 días hábiles → luego continuá con Paso 3
⚠️ Este paso es prerequisito. Sin empresa verificada, Meta no aprueba el acceso Tech Provider.
3
Crear / verificar la aplicación en Meta Developers
▼
  1. Entrá a developers.facebook.com/apps
  2. Si no tenés app creada: clic en "Crear aplicación" → tipo "Empresa" → agregar producto WhatsApp
  3. Si ya tenés la app: abrila y verificá que tenga el producto WhatsApp agregado en el panel izquierdo
4
Solicitar acceso Tech Provider — Embedded Signup
▼
  1. Dentro de tu app en Meta Developers → WhatsApp → Configuración
  2. Buscá la opción "Embedded Signup" o "Registro embebido"
  3. Solicitá Acceso avanzado para los permisos: whatsapp_business_management y whatsapp_business_messaging
  4. Meta revisa la solicitud: generalmente 1 a 3 días si la empresa ya está verificada
  5. Una vez aprobado, podés generar el flujo de Embedded Signup para tus clientes
⚠️ Este es el paso clave. Sin los permisos avanzados aprobados no podés hacer el onboarding de clientes.
5
Configurar el flujo de onboarding en n8n
▼

Una vez aprobado como Tech Provider, el flujo completo en n8n es:

Embedded Signup (web del cliente)
  → Webhook n8n recibe: WABA ID + Phone ID + Token temporal
  → Intercambiar token temporal por token permanente
  → Guardar credencial en n8n
  → Suscribir webhooks del cliente a su instancia n8n
  → Handshake Meta ↔ webhook (GET + token de verificación)
  → Mensajes del cliente llegan al webhook POST de su instancia

Plantillas de n8n necesarias (en orden):

1
Onboarding Finish

Recibe WABA ID + Phone ID al terminar el signup del cliente

2
Token Exchange

Intercambia el token temporal por el permanente y crea la credencial en n8n

3
Login Code

Registra el evento de login (sin guardar el token directamente en la tabla)

4
Subscribe Webhooks

Suscribe los mensajes del cliente a tu webhook principal

5
Overrides

Redirige los mensajes de cada cliente a su instancia de n8n específica

6
Verificar Coexistence en un número de cliente
▼
  1. Durante el Embedded Signup, al ingresar el número del cliente
  2. Si aparece ✅ palomita verde → elegible para Coexistence
  3. Si no aparece → el número puede tener mala reputación, ser nuevo, o el portafolio comercial tiene restricciones activas
📱 El cliente debe tener WhatsApp Business (no la versión normal)
💬 Debe tener historial de mensajes previo en ese número
✅ El portafolio comercial sin restricciones activas
⭐ Buena reputación del número con Meta
7
Separar cada cliente en su propia instancia de n8n
▼

Cada cliente debe tener su propia instancia de n8n. Así se hace el override:

  1. En la instancia del cliente, crear un workflow con webhook (GET + POST)
  2. Nombrar el path identificando al cliente: automatizacion-cliente-x
  3. Copiar la URL de producción del webhook (no la de test)
  4. En el workflow Overrides (instancia maestra) pegar: nombre del cliente, callback URL y un token de verificación inventado
  5. Publicar el workflow del cliente ANTES de ejecutar Overrides
  6. Ejecutar Overrides → Meta hace el handshake → mensajes quedan suscriptos a esa instancia
💡 Recomendación: cada cliente con su infraestructura separada para no mezclar datos y evitar problemas con Meta.

✅ Checklist de verificación

Mensajes de prueba llegando correctamente al webhook de n8n
