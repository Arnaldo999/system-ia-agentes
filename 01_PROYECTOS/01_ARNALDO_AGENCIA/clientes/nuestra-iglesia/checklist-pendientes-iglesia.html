📋 Checklist Pendientes — Iglesia Nueva Vida

Lista de información y archivos que el equipo de medios debe aportar para que el sistema funcione con su contenido real.
📅 Reunión sábado 10 AM 👤 Pastor Pablo ⚙ Por: Arnaldo Ayala
El sistema YA está construido y funcionando. Lo que ven hoy en la demo usa datos de muestra (12 versículos genéricos + 8 canciones populares de adoración latina). Para que el sistema sea de ustedes y refleje su identidad, necesitamos lo siguiente. Una vez recibido, cargamos todo del lado nuestro y queda 100% personalizado para el próximo culto.

📖 Versión bíblica que usan

El sistema permite cargar cualquier versión. Necesitamos saber cuál usan habitualmente para precargarla.

Opciones comunes: Reina-Valera 1960 (RV1960), Nueva Versión Internacional (NVI), Nueva Traducción Viviente (NTV), Dios Habla Hoy (DHH), Reina-Valera Contemporánea (RVC), otra.

¿Tienen el archivo digital? Word, PDF, JSON, base de datos, o solo está dentro del software actual.

🎵 Repertorio de canciones

Lista de canciones que cantan habitualmente en cultos. Por cada una necesitamos: título, autor, letra completa con estrofas y coros separados.

¿Tienen un cancionero ya armado (Word, PDF, app)? ¿O usan cada vez la app del músico?

¿Cuántas canciones tienen en repertorio activo? (10 / 30 / 100+)

¿Tienen licencia CCLI / SongSelect? Es importante para legalidad.

🎤 Grabaciones MP3 — versión propia de cada canción

Para que las letras avancen automáticamente sincronizadas con el grupo de cantos, necesitamos un MP3 grabado por su banda (no del autor original). Así el sistema sigue el tempo, las pausas y los arreglos exactos que ellos tocan.

¿La consola de audio tiene grabación USB? Esa es la mejor opción (audio limpio).

Si no, alcanza con un celular cerca del altavoz principal durante un ensayo.

Una vez grabado: 1 archivo MP3 por canción, sin editar. El sistema lo procesa con IA y genera los timestamps automáticos.

🖼 Logo en alta resolución

El logo actual que tenemos cargado es de baja resolución (~250x250 px). Para proyectarlo en pantalla gigante sin que se pixele necesitamos una versión más grande.

Ideal: archivo vector (.SVG, .AI, .PSD, .EPS) — escala a cualquier tamaño sin perder calidad.

Aceptable: PNG mínimo 1080x1080 px con fondo transparente.

¿Quién diseñó el logo? Capaz tienen los archivos originales.

🎨 Identidad visual de la iglesia

Para que el sistema se sienta de ustedes, necesitamos saber:

¿Tienen colores definidos? (ej: el magenta del logo + alguno secundario)

¿Tipografía oficial? Si no, elegimos una que combine.

¿Manual de marca o brand book? (Si no existe, lo armamos basado en el logo)

¿Hay alguna palabra/frase que los identifica? (lema, versículo distintivo)

📺 Material gráfico para anexos

El sistema permite mostrar imágenes y videos en la pantalla gigante entre cantos o durante anuncios. Útil para:

Flyer del culto del día / próximos eventos

Fotos del equipo pastoral, eventos, retiros

Videos cortos de testimonios, llamados a ofrenda, anuncios

¿Tienen carpeta o drive con material existente? Pueden ir cargándolo cuando lo tengan listo.

⚙ Información técnica del setup actual

Para garantizar que el sistema funcione bien con el equipo que ya tienen:

¿Qué PC usan para presentar? (marca, modelo, RAM, sistema operativo)

¿Cuántas salidas de video tiene? (HDMI, VGA) — ideal 2: una para el operador, otra para la pantalla gigante.

¿Cómo se conecta a la pantalla gigante? (HDMI directo, extensor, wireless)

¿La consola de audio tiene salida AUX o USB para que llegue audio limpio a la PC?

¿Streaming a YouTube/Facebook desde la misma PC u otra?

👥 Equipo de medios — quiénes operan

Para coordinar capacitaciones y entregar el sistema a personas concretas:

¿Cuántas personas están en el equipo de medios?

¿Quién opera la pantalla durante el culto? Nombre y contacto.

¿Rotan o siempre es la misma persona?

¿Qué nivel técnico tienen? (autodidactas / curso formal / programadores)

¿Días/horarios disponibles para una sesión de capacitación de 1 hora?
