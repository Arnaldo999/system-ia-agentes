## Proyecto solicitado — Ecosistema digital completo

A diferencia de [[maicol]] (1 marca, 1 vertical inmo) o [[cesar-posada]] (1 marca, vertical turismo), Grupo Back es el **primer cliente multi-marca de Arnaldo**. Las piezas del ecosistema (revisión 2026-04-27 con el alcance real):

| Pieza | Detalle |
|-------|---------|
| 🌐 **Sitio web multi-marca** | `grupoback.com` con home del grupo + 7 sub-páginas (`/rizoma`, `/misionerita`, `/patricios`, `/martina`, `/bocanada`, `/club-progreso`, `/mision-emprender`). Cada una con skin propio. Las últimas 2 son tipo blog. |
| 💬 **5 bots WhatsApp dedicados (1 nº por servicio)** | **Decisión confirmada 2026-04-27 tarde**: un número WhatsApp por negocio. Rizoma + Misionerita + Patricio's + La Martina + Bocanada. Club Progreso y Misión Emprender (blogs) no llevan bot. |
| 📊 **CRM multi-negocio** | Sidebar separando las 5 marcas con bot. **Ficha única de cliente** (cross-marca): si alguien alquila depto en Rizoma hoy y reserva mesa en Misionerita en 6 meses, es la misma persona. |
| 📅 **Cal.com multi-evento** | Eventos por marca: "Visita propiedad Rizoma" / "Reserva grupal Misionerita" / "Cita alquiler salón Patricio's" / "Reserva Martina" / etc. |

## Stack asignado (cliente Arnaldo)

| Componente | Servicio | Notas |
|------------|----------|-------|
| **Sitio web** | **WordPress** | Hosting profesional + dominio `grupoback.com` (pendiente OK Patricia). 7 sub-sitios bajo un mismo WordPress. |
| **Automatizaciones** | **n8n** | Motor de automatizaciones que conecta WhatsApp, Airtable, Cal.com y todos los servicios entre sí. Instancia en VPS Arnaldo. |

| **LLM** | OpenAI Arnaldo | |
| **Seguridad** | Capas de seguridad | HTTPS/SSL en todas las URLs, tokens protegidos, backups diarios automáticos, aislamiento físico. |

## Cláusula contractual especial — tratamiento de datos

Pendiente de confirmar enfoque exacto con Arnaldo, pero el contrato debe contemplar al menos:

- Aislamiento físico de datos del cliente (regla #0 del ecosistema)

- Confidencialidad (no mezclar con otros clientes, no usar para entrenar)

- Política de backups y retención

## Notas

- Es el **primer cliente multi-marca/multi-rubro** de Arnaldo. Si sale bien, queda como template para futuros grupos empresariales.
- La idea de "ficha única cross-marca" es similar al modelo de **persona única + roles** del [[wiki/entidades/crm-v2-modelo-robert|CRM v3 de Robert]] (`roles TEXT[]` con [comprador, inquilino, propietario]) — acá los roles son cross-marca [cliente-eventos, inquilino-inmo, comensal-misionerita, etc.].
- Por la complejidad cross-marca + ficha única, hay un argumento fuerte para **PostgreSQL dedicado tipo workspace** (regla #0 Postgres) en lugar de Airtable. Decisión pendiente.
